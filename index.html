📸 앱 사용법 (상세)

1단계 · API 키 · 템플릿 업로드

  1. Gemini API 키 칸에 위에서 발급받은 키 붙여넣기 → 🧪 테스트·저장 클릭
    • Gemini 서버에 실제 연결 테스트 후 문제 없으면 자동 저장됩니다.
    • 키가 잘못됐거나 할당량 초과면 바로 알려줘요.
  2. PAPS 템플릿 엑셀 업로드: 나이스에서 받은 학급별일괄입력_○○○.xlsx 파일을 드래그하거나 클릭해서 업로드
  3. 업로드 직후 측정종목 자동 인식 결과가 칩으로 뜹니다 (예: 왕복오래달리기(회), 악력(kg) 1차 오른쪽, ...)
🤖 모델은 gemini-2.5-pro로 고정되어 있습니다. 손글씨·복잡한 표 판독 정확도가 가장 높은 모델이에요. 속도보다 정확도를 우선해서 이걸 기본으로 썼습니다.
💡 이 앱은 학교 양식을 바꾸지 않아요. 헤더 그대로 읽어서 그 양식에 맞게 값만 채웁니다. 50m달리기를 쓰든 왕복오래달리기를 쓰든, 체지방률을 측정하든 안 하든 학교 양식을 100% 그대로 따라갑니다.

2단계 · 사진 업로드

  1. 반 선택: 업로드할 사진이 어느 반 측정기록인지 고르세요. 반이 섞이면 따로 올려야 합니다.
  2. 힌트(선택): AI에게 힌트가 될 한 줄 메모. 예:
    • 왼쪽 열이 1차, 오른쪽 열이 2차
    • 악력 왼손만 측정
    • 체육관 바닥에 놓고 찍은 사진
  3. 사진 선택/촬영: 여러 장 한 번에 OK. 손글씨든 인쇄된 기록표든 다 됩니다.

📷 정확한 판독을 위한 촬영 팁

  • 번호와 이름이 또렷하게 보이게 찍기 — AI는 번호로 학생을 매칭합니다. 번호가 잘리거나 흐리면 데이터가 엉뚱한 학생에게 들어갈 수 있어요.
  • 결시·미측정은 빈칸으로 두지 말고 / 또는 결 표시를 해주세요. 완전 빈칸이면 AI가 "위 학생 데이터를 아래로 당겨오는" 실수를 할 수 있어요. 사선 하나만 그어도 정확도가 크게 올라갑니다.
  • 표 제목·단위가 같이 보이게 찍기 (예: "악력(kg) 1차 右" 이런 헤더)
  • 기록지가 기울어지지 않게. 반듯이.
  • 그림자·반사광 피하기. 자연광 또는 균일한 조명 아래.
  • 손글씨면 가까이 크게. 흐릿하면 AI도 못 읽어요.
  • 한 장에 너무 많은 종목이 섞여 있으면 종목별로 나눠서 찍는 게 정확도 ↑
🚨 가장 흔한 실수: 결시 학생 칸을 그냥 비워두면, 그 다음 학생 기록이 위로 올라가서 잘못 매칭될 수 있어요. 꼭 / 표시를 해주세요. 만약 이런 일이 생기면 검수 카드에 🚨 이름 불일치 경고가 떠서 알려줍니다.

✍️ 측정기록지 작성 팁 (AI가 잘 읽도록)

아무리 좋은 AI여도 글씨가 흐리거나 작으면 못 읽습니다. 기록지 적을 때 다음을 지키면 정확도가 크게 올라가요:

  • 또박또박 크게 적기. 칸을 가득 채울 정도로 큼직하게. 흘려쓰면 AI도 사람도 못 읽어요.
  • 숫자는 하나씩 분리해서 쓰기. 예: 8 5 ❌ → 85 ✅ (붙여 쓰되 각 숫자가 명확히 구분되게)
  • 소수점은 점(.)으로 명확하게. 가운뎃점·따옴표·공백은 피하세요. 예: 23.5
  • 8과 9를 명확히 구분해 쓰기:
    • 8: 위·아래 두 동그라미를 모두 완전히 닫기
    • 9: 위 동그라미를 닫고 아래 꼬리를 길게 빼기
  • 0과 6, 1과 7도 헷갈릴 수 있어요:
    • 0은 동그라미만, 6은 위에 짧은 꼬리
    • 1은 직선만, 7은 위에 가로획
  • 지우개로 지운 흔적이 있으면 그 위에 또 쓰지 말고, 옆 빈 곳에 다시 쓰거나 줄을 긋고 새로 쓰기. 겹친 글씨는 AI가 못 읽어요.
  • 연필보다는 진한 펜(검은 볼펜·사인펜)으로. 연필 글씨는 사진에서 흐려져요.
💡 미리 대비: PAPS 측정 시작 전에 학생들에게 "또박또박 크게 적어달라"고 안내해두면, 입력 단계에서 들이는 시간을 크게 줄일 수 있어요.

3단계 · AI 분석 시작

  1. 🔍 AI 분석 시작 버튼 클릭
  2. 사진 한 장당 2~5초 소요. 완료되면 각 사진 옆에 감지된 종목이 초록 칩으로 표시:
    악력(kg) 1차 右 악력(kg) 1차 左 학생 21명
  3. 실패한 사진은 ❌ 에러 메시지가 뜹니다. 사진 다시 찍거나 힌트 추가 후 재시도.

4단계 · 검수 · 수정

분석이 끝나면 아래 표에 값이 자동으로 채워집니다. 눈으로 확인할 때 다음 색깔을 참고:

원본 (AI 안 건드림)
AI가 채운 값
수동 수정한 값
범위 밖 (소수점 의심 ⚠)

위쪽 신뢰도 집계 pill에 높음 / 보통 / 낮음 개수가 뜹니다. 낮음과 범위 밖(⚠)만 집중 검수하면 끝.

⚠ 범위 밖 자동 감지
AI가 23.5kg을 235kg으로 잘못 읽으면 (소수점 누락) → 빨간 테두리로 표시됩니다. 가장 흔한 실수를 자동으로 잡아줘요.

편리 기능:

  • 낮음만 보기 토글: 신뢰도 낮은 학생만 필터링
  • 수정 되돌리기: 수동 수정분 전부 초기화 (AI 값은 보존)
  • 반 바꾸기: 위에서 반 선택 바꾸면 그 반 데이터로 즉시 전환. 다른 반도 똑같이 작업 가능.

5단계 · 엑셀 다운로드

💾 엑셀 다운로드 버튼을 누르면 학급별일괄입력_YYYYMMDD_HHMM.xlsx 파일이 받아집니다.

  • 원본 양식 그대로 값만 채워진 파일입니다.
  • 값은 모두 순수 숫자로 저장 (단위 기호 없음) → 나이스 업로드 그대로 가능
  • 검수 중인 내용은 다운로드에 그대로 반영됩니다.

❓ 자주 묻는 질문

매번 엑셀 명단을 다시 올려야 하나요?

아니요. 최초 1회만 업로드하면 됩니다. 업로드한 학생 명단(번호·이름·반 정보)은 브라우저에 자동 저장되어, 다음 방문 시 자동으로 불러옵니다.

나이스에서 받은 학급별일괄입력이든 학년별일괄입력이든 모두 지원됩니다. 학년별로 올리면 그 안에 든 모든 반이 한 번에 등록돼요.

학생이 바뀌었거나 새 학기가 시작되면 🔄 새 명단 업로드 버튼으로 교체하세요.

※ 기록값(측정 결과)은 저장되지 않고 명단만 저장됩니다 — 새로 측정할 때마다 빈 상태에서 시작.

같은 반에 여러 종목을 나눠서 올려도 되나요?

네. 예를 들어 오늘은 악력 사진만 업로드 → 검수 → 다운받지 말고 그대로, 내일 50m달리기 사진 업로드 → 검수 → 다운로드. 같은 반이면 계속 누적됩니다.

단, 브라우저를 닫거나 새로고침하면 작업 내용이 사라져요. 작업 끝난 반은 다운로드 받아두세요.

여러 반을 한 번에 처리하려면?

반 하나 작업 끝내고 다운로드 → 반 선택 바꿔서 다음 반 사진 업로드. 혹은 브라우저 탭을 여러 개 열어서 각각 다른 반 작업해도 됩니다.

AI가 숫자를 잘못 읽어요

이미 최고 성능 모델(gemini-2.5-pro)이 고정이라 두 가지만 시도해보세요:

  1. 사진 다시 촬영: 가까이, 수평으로, 그림자 없이. 기울기·반사광이 가장 큰 원인.
  2. 힌트란에 문맥 추가: "왼쪽 1차, 오른쪽 2차" 같은 힌트가 큰 도움이 됩니다.

그래도 안 되면 해당 셀만 클릭해서 직접 수정. 빨간 테두리 셀(범위 밖)만 훑어도 대부분 잡혀요.

8을 9로 읽거나 9를 8로 읽어요 (가장 흔한 오인식)

손글씨에서 8과 9는 가장 헷갈리는 숫자입니다. 8의 위쪽 동그라미가 살짝만 열려도 9로 읽힐 수 있고, 9의 꼬리가 짧으면 8로 읽힐 수 있어요.

가장 확실한 해결책 — 측정기록지 작성 단계에서:

  • 또박또박 크게 쓰기. 작거나 흘려 쓴 숫자는 AI가 못 읽어요.
  • 8은 위·아래 두 동그라미를 모두 완전히 닫아서 쓰기.
  • 9는 위 동그라미를 닫고 아래 꼬리를 길게 빼기 (직선 또는 곡선).
  • 다른 헷갈리는 짝도 신경: 0(완전 동그라미) vs 6(위에 짧은 꼬리) / 1(직선) vs 7(가로획 있음)

이미 작성된 기록지의 경우:

  • 사진을 최대한 가까이·또렷하게 다시 촬영. 픽셀이 충분해야 AI가 구분합니다.
  • 분석 결과 표에서 8/9 들어간 값을 한 번 훑어보고, 의심되는 셀만 사진 보면서 직접 수정.
소수점이 자주 틀려요 (23.5 → 235 같은)

이 앱은 소수점 오인식을 자동으로 감지합니다. 악력이 235kg처럼 비현실적이면 빨간 테두리 + ⚠ 표시가 떠요. 해당 셀만 클릭해서 직접 수정하세요.

원천적으로 줄이려면: 기록지 숫자를 또렷하게 쓰고, 소수점을 명확히 찍어주세요 (가운뎃점 · 작은따옴표 ' 도 소수점으로 인식하지만 점이 가장 정확).

학생 번호가 잘못 매칭돼요

이 앱은 학생 번호 기준으로 엑셀에 기록합니다. 이름이 비슷해도 번호가 다르면 엉뚱한 학생에게 값이 들어가지 않도록 안전장치가 걸려 있어요.

AI가 번호를 잘못 읽었다면: 기록지에 번호가 또렷이 보이게 촬영, 또는 힌트란에 "사진에 번호가 행 왼쪽 끝에 있음" 같은 안내를 추가하세요.

API 키를 저장해도 새로 열면 또 넣어야 해요

다른 브라우저에서 열었거나, 시크릿 모드에서 사용했거나, 브라우저 설정에서 사이트 데이터 삭제를 했기 때문입니다. 같은 브라우저(일반 모드)에서 사용하면 저장됩니다.

데이터는 어디로 가나요? (개인정보 우려)

사진 → Google Gemini API로 전송되어 판독됩니다. Google 무료 등급은 입력이 모델 개선에 사용될 수 있다는 약관이 있어요 (유료 등급은 사용 안 함).

학생 개인정보(사진 속 이름 등) 보호가 중요한 경우:

  • 유료 등급(Tier 1 이상)으로 업그레이드 후 API 키 사용, 또는
  • 사진에서 이름을 가리고 번호만 보이게 촬영 후 업로드

이 앱 자체는 외부 서버로 아무것도 보내지 않습니다. 브라우저 ↔ Google API 직접 통신.

인터넷 없는 곳에서도 쓰나요?

Gemini API 호출이 필요하므로 인터넷 필수입니다. 체육관에서 바로 찍어서 처리하려면 핫스팟이나 LTE 연결이 있어야 해요.

학급별·학년별 일괄입력 외에 다른 양식도 되나요?

나이스(NEIS)의 학급별 일괄입력·학년별 일괄입력 두 양식 모두 그대로 지원됩니다. 첫 번째 시트 1행이 헤더인 구조라면, 학교마다 측정종목이 달라도(50m·팔굽혀펴기·스텝검사 등) 헤더를 자동 인식합니다.

완전히 다른 구조의 양식이 필요하면 @kimju.zip에 DM 주세요.
